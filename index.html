SQL Editor
Not a textarea with coloring.
A real editor.
Custom tokenizer with 612 recognized tokens. Context-aware autocomplete. Two-pass alias tracking. All running in vanilla JS — no CodeMirror, no Monaco, no libraries.
SQL Editor — idleflow
SELECT u.username, u.email, COUNT(o.id) AS order_count, SUM(o.total) AS lifetime_value, MAX(o.created_at) AS last_order FROM users u LEFT JOIN orders o ON o.user_id = u.id WHERE u.active = 1 AND u.created_at >= '2025-01-01' GROUP BY u.id HAVING order_count > 5 ORDER BY lifetime_value DESC LIMIT 25;
12 Token Types
Keywords, functions, types, tables, strings, numbers, comments, operators, identifiers, backticks, variables, constants — each distinctly styled per theme.
Smart Autocomplete
Tables after FROM, scoped columns after WHERE, dot-notation for u.email when there's a FROM users u. Not just a word list.
Alias Tracking
Two-pass analysis: first extract tables + aliases from FROM/JOIN, then re-classify matching identifiers throughout the entire query.
Features
Everything you need.
Nothing you don't.
No bloat. Every feature exists because someone needed it while working on a real project.
⚡
Inline Cell Editing
Click any cell to edit. Enter saves via AJAX. Tab jumps to the next. Large text auto-switches to a textarea. The page never reloads.
🔍
Search Across Tables
Enter any value — email, ID, date — and scan every table in the database. Results grouped by table with matched cells highlighted. The feature phpMyAdmin doesn't have.
⊞
ER Diagram
Interactive SVG with every table and FK relationship. Drag tables, zoom, pan. Force-directed auto-layout clusters related tables. Hover to trace relationships.
⏱
Query History
Every query saved to session with syntax highlighting, timing, row counts. Click to reload into the editor. Search, delete individual items, or clear all.
☐
Table Operations
Rename, copy (structure or with data), and drop tables from the database overview. Visual Create Table form with live SQL preview. Bulk select and delete rows.
⇅
Import & Export
Drag-and-drop SQL/CSV import with target mode for new databases. Export single tables or full dumps. Insert rows visually with type-aware form fields.
🔑
FK Visualization
Structure tab shows gold/blue borders on PK/FK columns, clickable relation links, dedicated FK tables with ON DELETE / ON UPDATE badges.
⌨
Keyboard Shortcuts
Press ? anywhere to see every shortcut. Ctrl+Enter to execute, Tab for autocomplete, Esc to cancel. Designed for keyboard-first workflows.
ER Diagram
See your schema.
Not just imagine it.
Every table, every column, every foreign key — rendered as an interactive SVG you can drag, zoom, and explore.
Force-Directed Layout
Click "Auto Layout" and a physics simulation runs 300 iterations: tables repel each other, FK edges attract, gravity holds it together. Connected tables cluster. Unrelated ones drift apart.
Drag, Zoom, Pan
Drag any table to reposition it. Scroll wheel zooms centered on your cursor. Click empty space to pan. "Fit" button auto-centers everything. Double-click a table to open it.
Hover to Trace
Hover a table and its FK lines brighten while everything else dims. Instantly see which tables are connected and through which columns — without reading a single line of SQL.
Search & History
Find anything.
Remember everything.
Two features that phpMyAdmin simply doesn't offer.
Search Across Tables
Enter a value — an email, an ID, a date — and DBForge scans every table in the database.

Results come back grouped by table, with matched columns highlighted in purple. Click through to browse the matching rows. Filter 39 tables in one search instead of opening each one manually.

Skips BLOB and BINARY columns automatically. Links to the Browse tab with the search filter pre-filled.
Query History
Every query you execute is saved to the session — SQL text, database, timing, row count, success/error.

Collapsible panel on the SQL tab with full syntax highlighting. Click an entry to reload it into the editor. Filter by keyword. Delete individual items with ×. Clear all with one click.

Duplicate suppression: running the same query twice updates the timestamp instead of creating a new entry.
